本版本版本完成注册业务操作，

上一个版本中，HttpRequest已经将请求中的参数解析完毕了
这里我们开始完成一个具体业务，用户注册

实现：
1. 为HttpRequest对应的三个属性添加get方法，便于业务处理类获取用户提交的参数，并且ClientHandler也不在根据请求对象中的
url作为处理请求的判断了（因为url可能含有参数）

2.在ClientHandler处理请求的部分，首先修改代码，将获取请求路径的内容从url换乘获取requestURI

3.添加一个新的分支判断，首先判断该请求是否为请求注册业务，如果是则实例化处理注册业务的类并调用其处理方法
若不是则走原有的操作（去webapps下寻找该静态资源）

4.创建一个新的包:com.webserver.servlet
使用这个包保存所有用于处理业务的类
在该包中新建用于处理用户注册业务的类：RegServlet并完成该类功能，这样一来第三步就可以调用这个类
处理注册了
